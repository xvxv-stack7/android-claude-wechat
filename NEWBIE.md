# 新手必读：AI 不是遥控器，是你的导航仪

## Claude Code 能做什么、不能做什么

装好后你面对的是一只"黑窗口里的AI"。它只能做两件事：

### 能做：给命令、出主意

AI会告诉你该敲什么命令、该怎么配置。所有技术决策、代码调试、错误排查——它会一步步指导你。

### 不能做：替你动手

AI不会帮你敲回车、不会帮你粘贴、不会帮你创建文件——除非你配好了后面的mimic屏幕控制。所有需要改系统、写入文件、下载安装的操作，**都是你在Termux里手动执行AI给你的命令**。

简单说：AI说"跑这句话"，你在黑窗口里粘贴回车。跑完了把输出发给AI，它继续指导下一步。

## 三条铁律

1. **所有命令都在Termux的黑窗口里跑**，不是发给微信聊天框
2. **AI给的命令是一行一行粘的**，不是AI自己动手
3. **跑完命令把结果告诉AI**——你不说我就不管，不会猜你跑完了没

## 新手最容易踩的坑

### 坑1：以为AI能直接操作手机

不能。AI没有手。它是你的技术顾问，不是你的遥控器。所有操作都要你自己在Termux里执行。

### 坑2：把命令发到微信里

不对。微信是跟AI聊天的，Termux是跑命令的。两条路分开——微信聊天窗口说"帮我写代码"，Termux窗口跑"npm install"。

### 坑3：DNS没配就开跑

Termux默认没DNS。装Claude Code之前先把DNS配了（DNS步骤在README里），否则npm下载必断。

### 坑4：中间断了就以为失败了

无代理方案npm下载经常断。断了重跑就行——命令是幂等的，跑多少次结果一样。

### 坑5：没看报错就跳过

跑命令之后屏幕最下面几行就是结果。有报错字样的（Error、failed、not found）截下来发给AI，别自己猜。
